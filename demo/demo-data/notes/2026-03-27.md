## API Refactor Working Session

Spent the afternoon pairing with @priya on the auth middleware extraction. This is part of [[task:migrate-auth-to-oauth2]].

### Current Architecture

The auth logic is scattered across 12 route handlers. Each one does its own token validation:

```typescript
// Before: duplicated in every handler
app.get("/api/users/:id", async (req, res) => {
  const token = req.headers.authorization?.split(" ")[1];
  if (!token) return res.status(401).json({ error: "Missing token" });
  const user = await validateToken(token);
  if (!user) return res.status(403).json({ error: "Invalid token" });
  // ... actual handler logic
});
```

### Target Architecture

Single middleware that handles all auth concerns:

```typescript
// After: clean separation
const authMiddleware = createAuthMiddleware({
  provider: "oauth2",
  tokenStore: redisClient,
  refreshWindow: 300, // seconds before expiry
});

app.get("/api/users/:id", authMiddleware, async (req, res) => {
  // req.user is guaranteed to be set
  const user = req.user;
  // ... handler logic only
});
```
